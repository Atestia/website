D. ASSET MANAGEMENT

Inventory, classification, disposal

D.1

Is an inventory of information assets maintained?

YES · All AWS resources tracked via Terraform state. SaaS subscriptions tracked in operations registry. Reviewed quarterly.

D.2

Are assets classified by sensitivity?

YES · Three-tier classification: Customer Data (highest), Internal Confidential, Public. Customer Data is encrypted with customer-held KMS for Enterprise tier.

D.3

Are mobile devices managed?

YES · Personnel laptops enrolled in MDM (Jamf for macOS, Intune for Windows). Disk encryption enforced. Remote wipe capability.

D.4

Is media disposal handled securely?

YES · Cloud-native operations. No physical media containing customer data. AWS handles physical disposal of decommissioned hardware per their SOC 2.

E. ACCESS CONTROL

Authentication, authorization, privileged access

E.1

Is multi-factor authentication required for administrative access?

YES · MFA required for all administrative actions. TOTP + hardware key supported. Enforced via SSO provider.

E.2

Are user accounts unique and individually attributable?

YES · No shared accounts. Service accounts have unique IDs and rotated credentials.

E.3

Is access granted based on principle of least privilege?

YES · Role-based access. Default deny. Privileged access requires Just-In-Time (JIT) elevation with audit logging.

E.4

Is privileged access logged and monitored?

YES · All admin actions logged to immutable audit log. CloudTrail + custom audit events. Reviewed weekly.

E.5

Are session timeouts enforced?

YES · 15-minute idle timeout for admin sessions. 1-hour absolute session limit.

E.6

Is password complexity enforced?

YES · Minimum 16 characters. NIST 800-63B-compliant. No periodic rotation (per NIST guidance). Compromised password checks via HaveIBeenPwned.

E.7

Is SSO supported for customer authentication?

YES · SAML 2.0 + SCIM for Enterprise tier. Tested with Okta, Auth0, Microsoft Entra ID, Ping Identity, Google Workspace.

F. CRYPTOGRAPHY

Encryption, key management

F.1

Is customer data encrypted in transit?

YES · TLS 1.2+ enforced. TLS 1.3 preferred. HSTS enabled. Certificate management via AWS ACM.

F.2

Is customer data encrypted at rest?

YES · AES-256-GCM. AWS KMS-managed by default. Customer-held CMK (BYOK) supported for Enterprise tier.

F.3

Are encryption keys managed securely?

YES · AWS KMS with FIPS 140-2 Level 3 HSMs. Customer-held CMK option for Enterprise. Annual key rotation. No keys in source code or configuration.

F.4

Are cryptographic algorithms NIST-approved?

YES · AES-256, SHA-256, RSA-2048+, ECDSA P-256+. No deprecated algorithms in use.

F.5

Is customer-held KMS (BYOK) supported?

YES · Customer holds master key in their AWS account. Atestia uses envelope encryption with per-record data keys. Customer revocation immediately removes Atestia access to historical data.

Personal data handling

P.1

P.2

YES · Atestia is architected to capture metadata, never raw payloads. Default retention is minimum necessary per regulation.

P.3

YES · Access, rectification, erasure, portability, objection rights honored per GDPR/CCPA. 30-day response SLA.

P.4
