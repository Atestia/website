Platform

The runtime evidence layer for regulated AI.

Atestia is an Anthropic Model Context Protocol (MCP) server. Drop it into any AI agent platform and every vendor model invocation, risk data lineage event, and agent reasoning chain is captured at runtime in production. Then generate examiner-ready packets on demand.

The 10 tools

LINEAGE CAPTURE

tier4.credit.evaluate

Capture a vendor model invocation for a credit decision. Returns lineage_id, vendor_attestation_uri, sr_26_2_packet_uri, bcbs_239_aggregation_id.

LINEAGE CAPTURE

tier4.lineage.bcbs239

Capture a BCBS 239 risk data lineage event. Source system → transformation → target report (CCAR-Y14A, DFAST, etc.).

LINEAGE CAPTURE

tier4.lineage.regb

Capture ECOA Regulation B adverse action lineage. Required for credit denials under 15 USC 1691.

LINEAGE CAPTURE

tier4.lineage.fetch

Retrieve a full lineage trail by lineage_id. Used by examiners and MRM analysts.

REASONING CAPTURE

tier4.reasoning.capture

Capture an agent reasoning chain — the THINKING flow. Observation → tool call → tool result → decision, with confidence and human-in-loop status.

PACKET GENERATOR

tier4.packet.occ-2013-29

Generate the OCC 2013-29 vendor questionnaire (47 fields, 7 sections). Examination evidence ready.

PACKET GENERATOR

tier4.packet.sr-26-2

SR 26-2 vendor attestation pack: model card, OCC questionnaire, lineage trail spec, monitoring spec, SOC 2 control matrix.

PACKET GENERATOR

tier4.packet.bcbs-239

BCBS 239 risk-data aggregation reference. 14 Basel principles mapped to your model inventory.

PACKET GENERATOR

tier4.packet.regb-notice

ECOA Reg B Adverse Action Notice. Customer-facing notice (12 CFR 1002.9) + 25-month compliance record (1002.12). CFPB enforcement contact embedded.

ECOSYSTEM

tier4.openlineage.export

Export a Tier-4 lineage record as an OpenLineage RunEvent. Publish to Marquez, Snowflake Horizon, Databricks Unity Catalog, Astronomer.

Architecture

One MCP server. Three layers. Click any block to expand.

Atestia sits between your agent platform and your storage layer. Every block below is a live component — click to see what it does and how to wire it.

Layer 1 · Agent platforms

BBedrock▼
AWS Bedrock AgentCore — Native MCP support. Used by BNY Mellon Eliza, JPMorgan Index AI, Goldman Athena Commercial. Adds Atestia as a tool in any agent definition.
CClaude Desktop▼
Anthropic Claude Desktop + MCP — Drop into claude_desktop_config.json. Used by 70+ MRM analyst teams for ad-hoc compliance queries.
LLangChain▼
LangChain TS + Python — Atestia tools register automatically as LangChain Tools. Used by Capital One, M&T, Regions for custom underwriting agents.
$Cursor▼
Cursor IDE — For developer-facing AI tools that touch regulated data. Used by Atestia-customer engineering teams to keep dev-time AI use within governance scope.
MCP stdio

OCC 2013-29 · SR 26-2 · BCBS 239 · ECOA Reg B — materialized PDF + JSON on demand. 4-hour SLA on Tier-1. Includes attestation hashes for examiner-defensible review.
